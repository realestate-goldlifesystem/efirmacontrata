"ACTÚA COMO UN EXPERTO EN UI/UX Y DESARROLLO FRONTEND PARA WEB-APPS PREMIUM.
Quiero que construyas esta interfaz bajo los siguientes estándares de diseño, responsividad y experiencia de usuario (UX):

1. RESPONSIVIDAD Y FLUIDEZ (El nuevo estándar):
- Utiliza funciones matemáticas de CSS como `clamp()` y unidades relativas (`vw`, `vh`, `rem`) para las tipografías, espaciados y márgenes. El texto y los márgenes deben crecer o encogerse fluidamente sin saltos bruscos.
- No dependas EXCLUSIVAMENTE de `clamp()`. Usa Media Queries estratégicas (`@media`) para cambiar radicalmente la estructura entre PC y Móvil cuando sea necesario (por ejemplo: cambiar una columna lateral en PC por un menú desplegable oculto en móvil para ahorrar espacio).

2. EXPERIENCIA MÓVIL NATIVA (UX App-like) - ¡CRÍTICO!:
- NUNCA uses `height: 100vh;` para contenedores de pantalla completa. Usa SIEMPRE `height: 100dvh;` (Dynamic Viewport Height) para evitar problemas con la barra de navegación del navegador móvil.
- Si creas una barra inferior (Bottom Bar) para acciones rápidas en móviles, usa SIEMPRE `position: fixed; bottom: 0; left: 0; right: 0;` para anclarla con seguridad absoluta, evitando que quede atrapada detrás de los botones físicos o digitales del sistema operativo (Android/iOS).
- A esa Bottom Bar flotante, agrégale SIEMPRE el padding de seguridad de dispositivos modernos: `padding-bottom: calc(1rem + env(safe-area-inset-bottom, 0px));`.
- Asegúrate de dejarle espacio al Bottom Bar en el contenedor principal usando un `padding-bottom` generoso, para que el último bloque de contenido no quede oculto detrás de la barra.

3. DISEÑO PREMIUM Y ESTÉTICA:
- Crea una estética moderna, limpia y armónica (piensa en un diseño Dark Mode premium con contrastes sutiles, como grises oscuros #111827, toques dorados/primarios vibrantes #fbbf24 y textos legibles).
- Evita el diseño aburrido. Usa bordes redondeados (`border-radius: 8px o 12px`), sombras sutiles (`box-shadow`), transiciones suaves en los botones (`transition: all 0.3s ease`) y efectos hover.
- Mantén la armonía en la disposición de los elementos. Usa `display: flex` con `gap` en lugar de márgenes aleatorios para que las listas, badges y botones nunca se desalineen ni generen saltos de línea feos si la pantalla se achica.

4. FLUJOS DE USUARIO LÓGICOS (Control de Rondas y Bloqueos):
- Si hay un proceso de envío, validación o formulario, diseña lógicas de bloqueo (UI Locking). Cuando el usuario realiza una acción, OCULTA los botones de acción y muestra en su lugar un mensaje claro de '⏳ Acción registrada, esperando...', para evitar clics duplicados o alteraciones que rompan la lógica del backend.

5. ESTRUCTURA DE PANELES Y MODALES (App-like Layout / Scroll Interno):
- Para paneles o modales complejos que ocupan casi toda la pantalla, NUNCA fuerces al usuario a hacer scroll global para encontrar el botón final.
- Emplea una "Maquetación Flexbox con Scroll Interno": El contenedor principal (`.modal-body` o wrapper) debe tener `display: flex; flex-direction: column; height: 100%; overflow: hidden;`.
- Los contenedores de botones maestros (acciones globales) deben quedar por fuera del flujo que se desplaza, anclados siempre al fondo.
- Usa `flex: 1` en el contenedor del centro (donde van las listas, datos o previsualizadores visuales) y un `min-height: 0` para que absorba todo el espacio dinámicamente y "empuje" los botones abajo.
- Si el contenido del medio tiene mucha información (formularios largos), ponle SOLO a ese contenedor interno un `overflow-y: auto`. Esto garantiza una experiencia idéntica a la de una app de escritorio: barras de herramientas fijas y contenido desplazable en el medio."
